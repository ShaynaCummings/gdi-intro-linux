What is Linux?

  • Linux: A free/open source operating system (OS) released by Linus Torvalds in 1991, as an alternative to Unix, a powerful (and proprietary) OS developed by AT&T
  • Popular as a server OS, and as a core for other OS' (incuding MacOS!)
  • Linux distribution (distro): A packaged version of Linux that you can install on your dektop or server (Unbuntu, Debian, Fedora, etc)

What is the Command Line?

  • Command Line (CLI): a text-based interface to your computer
  • Not specific to a particular OS
  • Not required in order to use Linux (some distros include GUIs)

However...
"Linux" is most often used to refer to the Linux command line

Why learn Linux Command Line?

  • Many software applications can only be installed/used via the command line
  • In software dev/research, servers are usually accessed/managed via the command line
  • Do so many more things...and do them faster!

What's in here?

Getting more specific

Options & Arguments

Syntax: [command] -[options] [arguments]


            ls -l Documents

Shows more info about files/folders in our current directory

How do I know what options/arguments are available?

Man Pages

man (manual)

Shows description, syntax and options and other fun facts for a command


            man ls

Moving around

cd (change directory)

Moves you to the directory that you specify


            cd documents

Moving around

Specifying files/folders

Path: Hierarchy of directories that leads to a folder/file

  • Relative paths start from your current location
  • Absolute paths start from a fixed location

Moving around

Path syntax:

  • . 'Here' (current directory)
  • ../ One level up from current directory
  • ~/ Home directory (for the current user)
  • / 'Root' directory (top-most directory)

Moving around

Relative paths

Moving downward from your current location
. / can be omitted in most commands


    cd ./foldername/foldername
    cd foldername/foldername

Moving around

Relative paths

Moving upward from your current location


    cd ../
    cd ../../
    cd ../foldername 
    cd ../../foldername

Moving around

Absolute Paths

Abolute paths start from ~/ (home) or / (root) and only move downward


    cd /Users/user/foldername
    cd ~/foldername

Hint: If you get lost, there's no place like ~/ (home)!

Review

We learned some great stuff about navigation!

  • pwd show current location
  • ls list files/folders in a directory
  • cd change directory
  • man shows info about a command, including all of its available options

Creating/Managing Folders & Files

Creating Folders

mkdir (make directory)

Creates a new (empty) folder


        mkdir newfolder

Folder & file names

Linux is picky!

  • Case sensitive
  • Avoid spaces (use dashes or underscores)
  • Avoid special characters

  • Specify file extensions

Creating Files

touch

Creates a new (empty) file


        touch newfile.txt

Copying Files

cp (copy)

Copies a file to a specified location

Syntax: cp [source] [destination]


        cp newfile.txt newfile2.txt

Copying Files

cp -r (copy, recursively)

Copies a folder and its contents to a specified location

Syntax: cp -r [source] [destination]


        cp -r newfolder newfolder2

Moving & renaming

mv (move)

Moves a file/folder (and its contents) to a specified location
OR renames it, if no location specified

Syntax: mv [source] [destination]


  mv newfile.txt newname.txt
  mv newfolder/newname.txt newfolder2/newname.txt

Deleting Files

rm (remove)

Removes a file


        rm newfile2.txt

Deleting Folders

rm -r (remove, recursively)

Removes a folder and its contents


        rm -r newfolder

Review

Essentials for wrangling files and folders:

  • mkdir create an empty folder
  • touch create an empty file
  • cp copy a file or folder (-r to copy folder)
  • mv move or rename a file or folder
  • rm remove a file or folder (-r to remove folder)

Editing files

Opening files

open

Opens a file in its default application (use -a option to open in a specific application)

Syntax: open [filename] -a [application]


        open fruit.txt -a textedit

Open can cause scripts/programs to run - don't use it on unknown files!

Command line text editors

  • Open/edit files right in your command line application
  • Keyboard only - no mouse/no GUI
  • Many available: vi, vim, nano, pico, emacs

Vi editor

  • Longstanding/popular editor; often the default on Linux systems


Check to see if vi is installed:


        which vi

Opening files in vi

vi

Opens an existing file in the vi text editor


        vi fruit.txt

Basic vi commands

  • i (insert mode) add text
  • esc (command mode) navigate, delete text
  • x delete characters to the right of the cursor
  • X delete characters to the left of the cursor
  • :w (write) save the file without quitting
  • :wq (write) save the file and quit
  • :q (quit) quit without saving
  • :q! (force quit) force quit without saving

Editing files in vi


    apple
    banana
    orange
    ~
    ~
    --INSERT--

Add some text to the file (i), then (esc) and quit (:q)

Creating new files with vi

Syntax: vi [filename]


        vi veggies.txt

Review

Essential commands for editing files:

  • open open a file in its default application
  • vi open a file in the vi text editor
  • i (vi) insert mode
  • esc (vi) command mode
  • :wq (vi) save and quit
  • :q (vi) quit

Searching

Searching for files/folders

find

Finds a file or folder

Syntax: find [location] [options] [search terms]


    find . "practice"
    find . -type d "practice"

Wildcards!

  • * match 0 or more characters
  • ? match 1 character
  • [ ] match range of characters

    find . "*.txt"
    find . "?ruit"
    find . "[a-k]*.txt"

Searching within files

grep

Finds text within files

Syntax: grep [options] [search terms] [file(s)] 


    grep "apple" fruit.txt
    grep "apple" practice/*
    grep -r "apple" ~

Search tips

  • Sytem-wide search: Use find/grep on your root directory to search your entire computer...but be careful! (can take a long time)
  • Permissions issues: find/grep only work on folders/files that you have permission to access (more on that soon!)
  • Cancel a search (or any command): control + c

Review

Essential search commands:

  • find searches for files and folders
  • grep searches for text within files

Practice

Put your sleuthing skills to use:

  1. Find all the folders in your home directory that begin with a vowel
  2. Choose a classmate's name and search your home directory for files containing that name

 

 

 

Permissions

Permission basics

  • Every file/folder has a set of permissions
  • Permissions define what each user can do with the file/folder, based on their role
  • Permissions can be changed by the file/folder owner or by a superadmin or 'root' user

Permission types

Linux has 3 permission types:

  • r (read) view a file/folder's content
  • w (write) edit a file/folder
  • x (execute) run a file (if it's a program/script) OR enter a folder

User roles

...and 4 user roles:

  • (u) user/owner the user who owns a file/folder
  • (g) group the user group that a file/folder belongs to
  • (o) others everyone who is not the owner or a group member
  • (a) all everyone in all the roles above

Checking permissions

Use the ls -l command we learned earlier
to view file/folder permissions

Changing file permissions

chmod

Add or remove permissions for a file

Syntax: chmod [role][+ or -][permission] [file] 


        chmod g+w fruit.txt

Changing folder permissions

chmod works for folders too, but doesn't affect folder content!

 

To change permissions for all files in a folder, use:

chmod [role][+ or -][permission] [parentfolder]/* 


        chmod g+w newproject/*

The root user

For security, certain actions can only be taken by a 'super admin' called 'root'

  • 'root' has has access to everything on a machine and can do anything with it
  • On your own machine, you can log in as 'root', but it's not a great idea
  • Instead, you can use sudo to temporarily become 'root'

Run commands as root

sudo (super user do)

Run any single command as the root user

Example: Edit permissions on files you don't own


        sudo chmod g+w newproject/*

Review!

Essential commands for managing permissions:

  • ls -l check permissions for a folder/file
  • chmod change permissions for a folder/file
  • sudo run a command as 'root'

    Accessing other machines via SSH

    Accessing other machines

    In addition to running commands on your own computer, you can run commands on other machines as well!

    But why would we do this?

    • Install software, make configuration changes, manage backups etc on a server
    • Transfer files between a server and your computer
    • Run code that requires a server environment

    Accessing other machines

    ssh (secure shell)

    Access another machine remotely, via an encrypted, access-controlled connection

    Syntax: ssh [user]@[location]

    
        ssh gdiboston@209.208.111.254

    Class SSH Login

     

    
 ssh gdiboston@209.208.111.254

 password: gdiboston2016!

    Where are we??

    Let's do some exploring
    using commands we learned earlier

    
    gdiboston@nerfherder: pwd
    gdiboston@nerfherder: ls -l
    gdiboston@nerfherder: ls /

    Transferring files

    scp (secure copy)

    Transfer files between 2 machines

    Transfer from local to remote:
    scp [source path] [user]@[location]:[destination path]

    
    scp fruit.txt gdiboston@209.208.111.254:~/

    Transferring files

    Transfer from remote to local:
    scp [user]@[location]:[source path] [destination path]

    
 scp gdiboston@209.208.111.254:~/fruit.txt .

    Ending SSH sessions

    exit

    Close an SSH connection

    
    gdiboston@nerfherder: exit

    Review!

    Essential commands for accessing other machines:

    • ssh log into another machine remotely
    • scp transfer files between machines
    • exit close an ssh connection
