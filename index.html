What is Linux?

Free and open source Unix-like operating system (OS) first released by Linus Torvalds in 1991

  • On the Desktop: A 'distro' is packaged version of Linux that you can install on your dektop, that usually involves a graphical user interface (GUI). Examples: Unbuntu, Debian, Fedora, Mint.
  • As a server OS: Ubuntu Server, Red Hat, CentOS
  • As a core for other Operating Systems: Android, Chrome OS

What is the Command Line?

  • Command Line Interface (CLI): a text-based interface that allows you to give commands to your computer
  • Not specific to a particular OS
  • Not required in order to use Linux (some distros, such as Ubuntu, include GUIs)

Why learn the Command Line?

  • Faster than navigating a UI
  • Allows you more granular control over your machine
  • Many software programs can only be installed/used via CLI
  • Servers (especially remote) are usually accessed and managed via CLI

1. Navigation

Where are we?

pwd (print working directory)

Shows your current location in the directory (file) structure


            pwd

What's in here?

ls (list)

Shows files/folders in our current directory


            ls

Options & Arguments

Syntax: [command] -[options] [arguments]


            ls -l Documents

In this example, adding an option of -l shows more info about files/folders in our current directory

What does that command mean??? Using Man Pages

man (manual)

Shows description, syntax, possible arguments, options, etc for a given command


            man ls

Program help

--help is an option baked into some programs to give you more info about how to use their commands


            ruby --help

Moving around

cd (change directory)

Moves you to the directory that you specify


            cd documents

File System Layout

In OSX, /home/ is /Users/

Moving around

Specifying files/folders

Path: Hierarchy of directories that leads to a folder/file

  • Relative paths start from your current location:
    Photos/grumpycat.jpg
  • Absolute paths start from a fixed location:
    /Users/lizl/Desktop/Photos/grumpycat.jpg

Moving around: path syntax

  • . 'Here' (current directory)
  • ../ One level up from current directory
  • ~/ Home directory (for the current user)
  • / 'Root' directory (top-most directory)

Moving around: relative paths

Moving downward from your current location
. / can be omitted in most commands


    cd ./foldername/foldername
    cd foldername/foldername

Moving around: relative paths

Moving upward from your current location


    cd ../
    cd ../../
    cd ../foldername 
    cd ../../foldername

Moving around: absolute paths

Abolute paths start from ~/ (home) or / (root) and only move downward


    cd /Users/user/foldername
    cd ~/foldername

Hint: If you get lost, there's no place like ~/

Tricks For Later

  • tab autocomplete: start typing a filename or folder, and hit tab to try to autocomplete it
  • hidden/system files: are preceded with a period (.) and you usually don't want to modify or delete these! But you can view them by running ls -a.
  • multiple terminal tabs/windows: you can open a new terminal tab/window by pressing cmd+T or cmd+N
  • clearing your terminal: you can clear your screen by typing clear
  • terminal history: type history, or hit the up or down arrow to scroll through all the commands you've already entered
  • change the look/feel of your terminal: this can be managed under the 'preferences' or 'settings' menu, or via your bash profile (more on this later)

Review

  • pwd: show current location
  • ls: list files/folders in a directory
  • cd: change directory
  • man: shows info about a command, including all of its available options

2. Creating/Managing Folders & Files

Creating Folders

mkdir (make directory)

Creates a new (empty) folder


        mkdir folder1

Folder & file names

Linux is picky!

  • On Linux, filenames/folders are case-sensitive so that filename.txt is different than FileName.txt -- but on Mac, they aren't!
  • Avoid spaces (use dashes or underscores)
  • Avoid special characters
  • Always specify file extensions

Creating Files

touch

Creates a new (empty) file


        touch file.txt

Copying Files

cp

copies a file to a specified location 


    cp [source]  [destination]
    cp file.txt  file-copy.txt
    cp file.txt  subfolder/file.txt

Copying Folders

cp -r (copy, recursively)

Copies a folder and its contents to a specified location


        cp -r [source] [destination]
        cp -r folder1  folder2

Moving & renaming

mv

Moves a file/folder (and its contents) to a specified location
OR renames it, if destination is in the same folder


    mv [source]          [destination]
    mv folder1/file.txt  folder2/file.txt
    mv file.txt          newname.txt

Deleting Files

rm

Removes a file


        rm newname.txt

Deleting Folders

rm -r (remove, recursively)

Removes a folder and its contents


        rm -r folder1

⚠ Running rm -rf will permanently delete a whole folder and its contents without prompting you. Use with caution!

Review

  • mkdir: create an empty folder
  • touch: create an empty file
  • cp: copy a file or folder (-r to copy folder)
  • mv: move or rename a file or folder
  • rm: remove a file or folder (-r to remove folder)

3. Editing files

Opening files

open

Opens a file in its default application (use -a option to open in a specific application)


        open [filename] -a [application]
        open fruit.txt  -a textedit

⚠ open can cause scripts/programs to run - don't use it on unknown files!

Command line text editors

  • Open, edit and save files right in your terminal
  • Keyboard only - no mouse/no GUI
  • Examples: vi, vim, nano, pico, emacs
  • Extensions, syntax highlighting, and plugins available

Vi editor

  • Longstanding command line text editor; sometimes set as default on Linux systems
  • Vim (another popular editor) is a superset of Vi. Everything you do in Vi can be done in Vim
  • Using Vi today since it's installed by default on Windows, Linux, and Mac


Check to see if/where vi is installed:


        which vi

Opening files in vi

vi

Opens an existing file in the vi text editor


        vi fruit.txt

Basic vi commands

  • i (insert mode) add text
  • esc (command mode) navigate, delete text
  • x delete characters to the right of the cursor
  • X delete characters to the left of the cursor
  • :w (write) save the file without quitting
  • :wq (write) save the file and quit
  • :q (quit) quit without saving
  • :q! (force quit) force quit without saving

Editing files in vi


    apple
    banana
    orange
    ~
    ~
    --INSERT--

Add some text to the file (i), then (esc) and quit (:q)

Creating new files with vi


            vi [filename]
            vi veggies.txt

Combining files

cat (concatenate)

Combine the contents of multiple files into one file


  cat [file1]   [file2]     > [newfilename]
  cat fruit.txt veggies.txt > produce.txt

Review

  • open open a file in its default application
  • vi open a file in the vi text editor
  • i (vi) insert mode
  • esc (vi) command mode
  • :wq (vi) save and quit
  • :q (vi) quit

4. Searching

Searching for files/folders

find a file or folder


  find [location] [options] [search terms]
  find . "fruit.txt"
  find . -type d -name "fruit.txt"

Wildcards!

  • * match 0 or more characters
  • ? match 1 character
  • [ ] match range of characters

    find . -name "*.txt"
    find . -name "?ruit"
    find . -name "[a-k]*.txt"

Searching within files

grep

Finds text within files


  grep [options] [search terms] [file(s)]
  grep "apple" fruit.txt
  grep "apple" practice/*
  grep -r "apple" ~

Search tips

  • Sytem-wide search: Use find/grep on your root directory to search your entire computer...but be careful! (can take a long time)
  • Permissions issues: find and grep only work on folders/files that you have permission to access (more on that soon)
  • Cancel a search (or any command): control+c

Review

  • find: searches for files and folders
  • grep: searches for text within files

Practice

  1. Find all the folders in your home directory that begin with a vowel
  2. Choose a classmate's name and search your home directory for files containing that name

 

 

 

5. Permissions

Permission basics

  • Every file/folder has a set of permissions
  • Permissions define what each user can do with the file/folder, based on their role
  • Permissions can be changed by the file/folder owner or by a superadmin or 'root' user

Linux has 3 permission types

  • r (read): view a file/folder's content
  • w (write): edit a file/folder
  • x (execute): run a file (if it's a program/script) OR enter a folder

Linux has 4 user roles

  • (u) user/owner: the user who owns a file/folder
  • (g) group: the user group that a file/folder belongs to
  • (o) others: everyone who is not the owner or a group member
  • (a) all: everyone in all the roles above

Checking permissions

Use ls -l (command we learned earlier) to view file/folder permissions

Changing file permissions

chmod

Add or remove permissions for a file


  chmod [role][+ or -][permission] [file]
  chmod g+w fruit.txt

Changing folder permissions

chmod works for folder permissions too, but doesn't affect all the files inside of them

 

To change permissions for all files in a folder: 


chmod [role][+ or -][permission] [parentfolder]/*
chmod g+w newproject/*

The root user

For security, certain actions can only be taken by a 'super admin' called 'root'

  • 'root' has has access to everything on a machine and can do anything with it
  • On your own machine, you can log in as 'root', but it's not a great idea
  • Instead, you can use sudo to temporarily become 'root' if needed
  • ⚠ Avoid using sudo when you install programs if possible

Run commands as root

sudo (super user do)

Run any single command as the root user -- for example, edit permissions on files you don't own


        sudo chmod g+w newproject/*

Review!

Essential commands for managing permissions:

  • ls -l: view permissions for a folder/file
  • chmod: change permissions for a folder/file
  • sudo: run a command as 'root'

    • Practice

    6. Accessing other machines via SSH

    Accessing other machines

    In addition to running commands on your own computer, you can run commands on other machines as well! We will use SSH (Secure Shell) to do this.

    But *why* would we do this?

    • Install software, make configuration changes, manage backups, etc on a server
    • Transfer files between a server and your computer
    • Run code that requires a server environment
    • ⚠ Only ssh into machines that you trust

    Accessing other machines

    ssh (secure shell)

    Access another machine remotely, via an encrypted, access-controlled connection

    
        ssh [user]@[location]
        ssh gdiboston@209.208.111.254

    Class SSH Login

     

    
 ssh gdiboston@209.208.111.254

 password: gdiboston2016!

    Where are we??

    Let's do some exploring using commands we learned earlier

    
    gdiboston@nerfherder: pwd
    gdiboston@nerfherder: ls -l
    gdiboston@nerfherder: ls /

    Ending SSH sessions

    exit

    Closes the SSH connection with the other machine

    
    gdiboston@nerfherder: exit

    Transferring files: local to remote

    scp (secure copy)

    Securely transfer files between 2 computers - from local machine to remote

    
scp [source path] [user]@[location]:[destination path]
scp fruit.txt gdiboston@209.208.111.254:~/

    Transferring files: remote to local

    
scp [user]@[location]:[source path] [destination path]
scp gdiboston@209.208.111.254:~/fruit.txt .

    Review!

    Essential commands for accessing other machines:

    • ssh: log into another machine remotely
    • scp: transfer files between machines
    • exit: close an ssh connection
